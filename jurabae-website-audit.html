🔍 SEO & Metadata
6 issues
Meta description is a single word "Breathable" HIGH
This is the snippet Google displays in search results. It should be ~155 chars describing what Jurabae sells. Recommended: "Jurabae - bold, graphic streetwear socks made in India. Terry cotton, all-day comfort. Shop coffee, matcha & more designs."
Page title is "Home - Jurabae" HIGH
Generic and keyword-free. Should be: "Jurabae | Graphic Streetwear Socks - Breathable Terry Cotton" to target search terms and improve click-through rate from search.
No og:image tag set MEDIUM
When someone shares the link on WhatsApp or Instagram, it won't show a preview image. Critical for a Gen Z brand relying on social sharing. Add a high-quality 1200×630px og:image.
Twitter card set to "summary_large_image" but no og:image exists MEDIUM
The card will render blank on Twitter/X previews. Fix by adding og:image - this resolves both the og and twitter card issues simultaneously.
No structured data (schema.org) HIGH
No Product, Organization, or BreadcrumbList schema found. Adding product schema would enable rich results (star ratings, price in Google) - massive uplift for a small brand competing against larger retailers.
Canonical URL correctly set ✓
canonical: https://jurabae.com/ - No duplicate content risk. Robots meta is also correctly set to index, follow, max-image-preview:large.
🖥 UX & Conversion
8 issues
Duplicate navigation menus rendered in HTML HIGH
The nav menu appears 3 times in the source (desktop, mobile, secondary mobile). Bloats HTML, confuses screen readers, and can cause inconsistency if one version isn't updated.
Hero CTA says "Shop Coffee" for a sock store HIGH
The hero headline uses a coffee metaphor ("Brew Life On Your Feet") but the CTA says "Shop Coffee." This messaging mismatch is confusing on first impression. The concept needs clearer bridging to the product - change to "Shop Now" or "Shop Socks".
Marquee/ticker text duplicated 10+ times inline MEDIUM
The "Free Shipping · WELCOME10" ticker text is pasted many times in raw HTML to simulate a scrolling marquee. Use a proper CSS animation loop instead - much cleaner markup.
No size guide visible on homepage MEDIUM
Socks have size variants. No size guide link appears on the homepage or in the product section. This is a friction point before purchase, especially for first-time buyers.
Only 2 customer reviews visible MEDIUM
Priyal and Kanishka are the only testimonials. No star ratings, no review count. For a new brand, social proof volume matters enormously. Aim for at least 10–15 visible reviews with photos.
Two conflicting phone numbers in footer HIGH
+91 95095 41626 appears in the header/footer section while +91 95095 42626 appears in the bottom footer strip - a one-digit difference. Verify the correct number immediately before customers call the wrong one.
Welcome popup with WELCOME10 discount code ✓
First-visit popup with a 10% off code is a proven conversion driver. Well implemented.
"Track Your Order" in main nav ✓
Reduces post-purchase anxiety and support tickets. Excellent inclusion for a new D2C brand.

⚙️ Technical & Performance
7 issues
Elementor 4.0.5 - verify it's the latest version HIGH
Elementor updates frequently for security patches. Always run the latest version on a live e-commerce site handling customer payments and personal data.
No image lazy loading confirmed HIGH
The homepage loads 15+ images including large lifestyle shots. Without lazy loading this severely impacts LCP (Largest Contentful Paint) and Core Web Vitals scores. Elementor has a native lazy load toggle - ensure it's enabled in Settings → Performance.
Images are JPEGs/PNGs - not WebP HIGH
File paths show .jpeg and .png files throughout. Converting to WebP typically reduces image file size by 30–50% with no visible quality loss. Use Smush, ShortPixel, or Imagify plugin for bulk conversion on WordPress.
Inconsistent image optimisation - WebP only in some sections MEDIUM
The Instagram gallery section uses .webp images but hero and product sections use .png/.jpeg. Apply a consistent optimisation pass across all uploads.
Elementor CSS likely render-blocking MEDIUM
css_print_method-external means Elementor writes separate CSS files loaded on every page. Enable the "Improved Asset Loading" experiment in Elementor → Settings → Experiments to only load CSS for widgets actually present on each page.
No standard favicon / apple-touch-icon MEDIUM
Only msapplication-TileImage is set. Add a standard favicon.ico and apple-touch-icon (180×180px) for iOS home screen bookmarks. These are basic brand assets for any D2C site.
Google Fonts loading with font-display: swap ✓
font_display-swap in the generator meta prevents Flash of Invisible Text (FOIT) during font loading. Correct implementation.

Core Web Vitals are likely failing on mobile HIGH
15+ unoptimised images, render-blocking Elementor CSS, and no lazy loading puts the site well above Google's recommended LCP threshold of 2.5s. This directly hurts Google ranking for competitive keywords like "graphic socks India" and "streetwear socks."
Fix order: WebP conversion - lazy load - Elementor asset loading HIGH
Converting images to WebP alone can cut page weight by 40-50%. Enable Elementor lazy load. Then turn on the "Improved Asset Loading" experiment in Elementor settings. These three steps together can push the mobile score above 70.

No caching plugin is evident. WP Rocket or LiteSpeed Cache would serve pre-built HTML pages instead of running PHP on every visit - significantly cutting TTFB (Time to First Byte).
📱 Mobile UX Review
7 issues
No blog / content page exists HIGH
Jurabae has zero blog or editorial content. This is a critical missed opportunity - a blog with content like "How to style graphic socks," "Best streetwear socks 2026," or "Matcha culture and fashion" would drive organic search traffic, build brand authority, and support AEO/GEO (AI Engine Optimization / Generative Engine Optimization). Without content, the site cannot rank for informational queries or appear in AI-generated answers.
3 overlapping navigation menus on mobile HIGH
Three separate nav menus are rendered in the HTML. On mobile, this creates layout conflicts and excess DOM weight. Only one mobile nav should exist - typically a hamburger that reveals a full-screen overlay menu.
Ticker bar is duplicated 10+ times - painful on mobile HIGH
The "Free Shipping / WELCOME10" announcement bar repeats the text 10+ times inline to fake a marquee scroll. On mobile this causes layout flash and excess content paint. Replace with a single CSS animation marquee.
Product grid likely too dense on small screens MEDIUM
The homepage product section shows multiple cards in a grid. On screens under 375px (iPhone SE, budget Android), these cards can become too narrow for comfortable tapping. Ensure the grid drops to 1 column below 400px and add padding to Add to Cart buttons (min 44x44px tap target).
Hero text readability on mobile MEDIUM
Large display text ("Brew Life On Your Feet") may overflow or stack awkwardly on mobile viewports. Verify line breaks, contrast ratio, and that the CTA button is full-width and thumb-friendly on mobile.
Instagram gallery images too large on mobile MEDIUM
The gallery section loads full-size lifestyle images without responsive srcset attributes. On mobile this wastes bandwidth and slows perceived load. Add srcset or use a responsive image gallery plugin.
Viewport meta tag is correctly set ✓
meta viewport="width=device-width, initial-scale=1" is present. The site will not render zoomed-out on mobile. ✓
